# 极简库存管理系统 - UI/交互设计文档

## 1. 设计原则

### 1.1 核心设计理念
- **简洁高效**：摒弃冗余元素，聚焦核心功能，让用户快速完成操作
- **直观易用**：符合Windows桌面应用的操作习惯，降低学习成本
- **专业稳重**：采用商务风格设计，体现数据管理系统的专业性

### 1.2 设计约束
- 适配Windows 10及以上版本
- 支持不同分辨率屏幕（1024x768及以上）
- 单次出入库操作≤3步

## 2. 界面布局规范

### 2.1 整体布局结构
采用经典Windows桌面应用布局：
```
┌─────────────────────────────────────────────────────────────────┐
│  顶部菜单栏（文件、编辑、帮助）                                  │
├─────────────┬───────────────────────────────────────────────────┤
│  左侧导航栏  │                                                 │
│  ├─ 商品管理 │                                                 │
│  ├─ 入库管理 │                    主内容区                      │
│  ├─ 出库管理 │                                                 │
│  ├─ 库存查询 │                                                 │
│  ├─ 台账报表 │                                                 │
│  └─ 账号管理 │                                                 │
└─────────────┴───────────────────────────────────────────────────┘
```

### 2.2 布局参数
- 左侧导航栏宽度：180px
- 顶部菜单栏高度：30px
- 主内容区边距：16px
- 控件间距：8px/16px/24px

## 6. 交互流程设计

### 6.1 入库流程
```mermaid
flowchart TD
    A[用户进入入库页面] --> B[选择商品]
    B --> C[输入入库数量]
    C --> D{数量是否有效?}
    D -->|是| E[填写备注]
    D -->|否| F[弹窗提示: 请输入有效数量]
    F --> C
    E --> G[点击确认入库]
    G --> H[系统更新库存]
    H --> I[系统记录入库流水]
    I --> J[弹窗提示: 入库成功]
    J --> K[刷新页面数据]
```

### 6.2 出库流程
```mermaid
flowchart TD
    A[用户进入出库页面] --> B[选择商品]
    B --> C[输入出库数量]
    C --> D{数量是否有效?}
    D -->|否| E[弹窗提示: 请输入有效数量]
    E --> C
    D -->|是| F[系统校验库存]
    F --> G{库存是否充足?}
    G -->|否| H[弹窗提示: 库存不足]
    H --> C
    G -->|是| I[填写备注]
    I --> J[点击确认出库]
    J --> K[系统更新库存]
    K --> L[系统记录出库流水]
    L --> M[弹窗提示: 出库成功]
    M --> N[刷新页面数据]
```

### 6.3 登录流程
```mermaid
flowchart TD
    A[用户打开应用] --> B[显示登录页面]
    B --> C[输入用户名]
    C --> D[输入密码]
    D --> E[点击登录]
    E --> F[系统验证账号密码]
    F --> G{验证是否成功?}
    G -->|否| H[弹窗提示: 用户名或密码错误]
    H --> C
    G -->|是| I[记录登录状态]
    I --> J[跳转到主界面]
```
